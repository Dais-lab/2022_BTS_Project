Hyperparameter

Epoch

- forward pass : 파라미터를 사용하여 입력부터 출력까지의 각 계층의 weight를 계산하는 과정을 거치는 순방향 패스

- backward pass : forward pass를 반대로 거슬러 올라가며 다시 한번 계산 과정을 거쳐 기존의 weight를 수정하는 역방향 패스

→ 이 전체 dataset에 대해 해당 과정(forwrad pass + backward pass)이 완료되면 한 번의 epoch가 진행됐다고 볼 수 있음

→ epoch 값이 너무 작다면 underfitting, 너무 크다면 overfitting이 발생할 확률이 높음

Batch size & Iteration

→ 메모리의 한계와 속도 저하 때문에 대부분의 경우 한 번의 epoch에서 모든 데이터를 한꺼번에 집어넣을 수는 없음

→ 그래서 데이터를 나눠주게 되는데 이때 몇 번 나눠주는가를 iteration, 각 iteration마다 주는 data size를 batch size라고 함

Learning rate

- learning rate가 큰 경우 : 데이터가 무질서하게 이탈하며, 최저점에 수렴하지 못함

- learning rate가 작은 경우 : 학습시간이 매우 오래 걸리며, 최저점에 도달하지 못함

Color

→ 데이터마다 학습할 때, 컬러/흑백 결과의 차이가 나타날 수 있음

Width

→ 모델마다 기본적으로 들어가는 standard size가 있음

→ 이미지마다 모양이 다 다르기 때문에(모양이 변형되면 결과가 달라질 수 있음) 그 모양을 변형할지에 대한 선택지를 줌

→ 크기에 따라 학습할 때 걸리는 시간의 차이가 있을 수 있음
